VB Control Manager

Features

 

 

Introduction

VB Control Manager is an ActiveX control to allow the user to dock controls and resize, move and show/hide them at run-time.

This form demonstrates VB Control Manager's properties, methods, events and its two collection properties.

 

Properties


1.   ActiveColor

      Returns/sets the background color used to display a splitter when the user moves it in none live update mode.


2.   BackColor

      Returns/sets the background color used to display all splitters.


3.   ClipCursor

      Returns/sets a value that determines whether the mouse pointer is confined to the virtual splitter minimum and maximum x- and y-coordinate when the user moves a splitter.


4.   Enable

      Returns/sets a value that determines whether all splitters are movable.


5.   Fill Container

      Returns/sets a value that determines whether the ActiveX Control (along with all controls inside it) will automatically adjust its size to fill-up its container with respect to the margin properties.


6.   LiveUpdate

      Returns/sets a value that determines whether the controls should be resized as a splitter is moved.


7.   MarginBottom

      Returns/sets the bottom margin of the ActiveX Control from its container.


8.   MarginLeft

      Returns/sets the left margin of the ActiveX Control from its container.


9.   MarginRight

      Returns/sets the right margin of the ActiveX Control from its container.


10.  MarginTop

      Returns/sets the top margin of the ActiveX Control from its container.


11.  Size

      Returns/sets the size of all splitters.


12.  TitleBar_CloseVisible

      Returns/sets a value that determines whether a close button in all control title bars is visible.


13.  TitleBar_Height

      Returns/sets the height of all control title bars.


14.  TitleBar_Visible

      Returns/sets a value that determines whether all control title bars are visible.

 

Methods


1.   CloseControl

      Closes (hides) a control.


      Syntax  :  Object.CloseControl IdControl [, Success]

      Parts     :  Object      -  Required. An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  Required. A value that uniquely identifies the control the developer want to close.

                     Success    -  Optional. A returned value that determines whether the Close method is successful.


2.   MoveControl

      Moves a control to certain area.


      Syntax  :  Object.MoveControl IdControlSource, MoveTo [, IdControlDestination, IdSplitterDestination, Success]

Parts     :  Object                      -  Required. An object expression that evaluates to a VB Control Manager object.

               IdControlSource         -  Required. A value that uniquely identifies the source control the developer want to move.

               MoveTo                     -  Required. A value indicating the area type where the source control will be moved to.

               IdControlDestination   -  Required only if the area type indicated by MoveTo is a control. A value that uniquely identifies the destination control the developer want to move the source control to.

               IdSplitterDestination   -  Required only if the are type indicated by MoveTo is a splitter. A value that uniquely identifies the splitter the developer want to move the source control to.

               Success                    -  Optional. A returned value that determines whether the MoveControl method is successful.


3.   MoveSplitter

      Moves a splitter to the specified x- or y- (depending on the splitter's Orientation property) coordinate.


      Syntax  :  Object.MoveSplitterIdSplitter, MoveTo

      Parts     :  Object      -  Required. An object expression that evaluates to a VB Control Manager object.

                     IdSplitter  -  Required. A value that uniquely identifies the splitter the developer want to move.

                     MoveTo    -  An integer value that specifies the x- or y- coordinate (depending on the splitter's Orientation property) where the splitter will be moved.


4.   OpenControl

      Opens (shows) a control and docks it to the ActiveX Control.


      Syntax  :  Object.OpenControl IdControl [, MaintainSize, Success]

      Parts     :  Object          -  Required. An object expression that evaluates to a VB Control Manager object.

                     IdControl      -  Required. A value that uniquely identifies the control the developer want to open.

                     MaintainSize  -  Optional. A value that determines whether the method will not change the size and position of the control the developer want to open. The default value is false.

                     Success        -  Optional. A returned value that determines whether the OpenControl method is successful.

 

Events


1.   ControlAfterClose

      Occurs when a control has just been closed by the user.

 

      Syntax  :  Private Sub Object_ControlAfterClose(ByVal IdControl As Long)

      Parts     :  Object      -  An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  A value that uniquely identifies the control that has just been closed by the user.

 

2.   ControlBeforeClose

      Occurs after the user presses a close button of certain control and before the control is closed.

 

      Syntax  :  Private Sub Object_ControlBeforeClose(ByVal IdControl As Long, Cancel as Boolean)

      Parts     :  Object      -  An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  A value that uniquely identifies the control that about to be closed.

                     Cancel      -  A boolean. Setting this argument to true stops the control from closing.

 

3.   ControlMove

      Occurs when the user is moving a control.

 

      Syntax  :  Private Sub Object_ControlMove(ByVal IdControl As Long, ByVal Shift As Integer, ByVal Left As Long, ByVal Top As Long, ByVal Width As Long, ByVal Height As Long)

      Parts     :  Object      -  An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  A value that uniquely identifies the control that is being moved by the user.

                     Shift         -  An integer that corresponds to the state of the SHIFT, CTRL, and ALT keys.

                     Left          -  An integer indicating the x-coordinate for the left edge of the current position of the rectangle that represents the moving control.

                     Top          -  An integer indicating the y-coordinate for the top edge of the current position of the rectangle that represents the moving control.

                     Width       -  An integer indicating the current width of the rectangle that represents the moving control.

                     Height      -  An integer indicating the current height of the rectangle that represents the moving control.

                    

4.   ControlMoveBegin

      Occurs when the user is about to move a control, i.e. the first time the rectangle that represents the moving control occurs.

 

      Syntax  :  Private Sub Object_ControlMoveBegin(ByVal IdControl As Long, ByVal Shift As Integer)

      Parts     :  Object      -  An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  A value that uniquely identifies the control that is about to be moved by the user.

                     Shift         -  An integer that corresponds to the state of the SHIFT, CTRL, and ALT keys.

 

5.   ControlMoveEnd

      Occurs when the user is finished moving a control, i.e. when the rectangle that represents the moving control disappears.

 

      Syntax  :  Private Sub Object_ControlMoveEnd(ByVal IdControl As Long, ByVal Shift As Integer, ByVal Moved As Boolean)

      Parts     :  Object      -  An object expression that evaluates to a VB Control Manager object.

                     IdControl  -  A value that uniquely identifies the control that has just been moved by the user.

                     Shift         -  An integer that corresponds to the state of the SHIFT, CTRL, and ALT keys.

                     Moved      -  A value that determines whether the control is moved.

Controls Collection

A Controls collection in VB Control Manager is a collection whose elements represent each virtual control in a Control Manager object. See the description of Control object in the next section below for the explanation of virtual controls.

The Controls collection has two properties: Count that specifies the number of elements in the collection and Item that returns a member of the collection. See reference for Collection Object in MSDN Library for the description of how to use the properties.

The Controls collection also has one method, IsExist, to determine whether certain index that uniquely identifies a virtual control exist in the collection.

 

Control Object

A Control object in VB Control Manager is a virtual control used to represent a real control (like TextBox control, Button control, etc.).

Instead of using real controls properties directly, VB Control Manager saves the necessary properties plus several custom properties in virtual controls. Furthermore, VB Control Manager works with these virtual controls to manipulate its position, size, etc. and then write it back to the real controls.

 

The Control object has 20 properties as follows:

 

1.   Bottom (read-only)

      Returns the distance between the internal bottom edge of the virtual control and the top edge of the related Control Manager object.


2.   Closed (read-only)

      Returns a value that determines whether the virtual control is closed.


3.   Height (read-only)

      Returns the height of the virtual control.


4.   Id (read-only)

      Returns a value that uniquely identifies the virtual control.


5.   IdSplBottom (read-only)

      Returns a value that uniquely identifies the virtual splitter which its movement will change the value of the virtual control's Bottom property.


6.   IdSplLeft (read-only)

      Returns a value that uniquely identifies the virtual splitter which its movement will change the value of the virtual control's Left property.


7.   IdSplRight (read-only)

      Returns a value that uniquely identifies the virtual splitter which its movement will change the value of the virtual control's Right property.


8.   IdSplTop (read-only)

      Returns a value that uniquely identifies the virtual splitter which its movement will change the value of the virtual control's Top property.


9.   Left (read-only)

      Returns the distance between the internal left edge of the virtual control and the left edge of the related Control Manager object.


10.  MinHeight (read-only)

      Returns the minimum height of the virtual control.


11.  MinWidth (read-only)

      Returns the minimum width of the virtual control.


12.  Name (read-only)

      Returns the name of the real control that the virtual control represents.


13.  Right (read-only)

      Returns the distance between the internal right edge of the virtual control and the left edge of the related Control Manager object.


14.  TitleBar_CloseVisible

      Returns/sets a value that determines whether a close button in the virtual control title bar is visible.


15.  TitleBar_Height (read-only)

      Returns the height of the virtual control title bars.


16.  TitleBar_Visible

      Returns/sets a value that determines whether the virtual control title bar is visible.


17.  Top (read-only)

      Returns the distance between the internal top edge of the virtual control and the top edge of the related Control Manager object.


18.  Width (read-only)

      Returns the width of the virtual control.


19.  Xc (read-only)

      Returns the x-coordinate of the virtual control center.


20.  Yc (read-only)

      Returns the y-coordinate of the virtual control center.

 

Splitters Collection

A Splitters collection in VB Control Manager is a collection whose elements represent each virtual splitter in a Control Manager object. See the description of Splitter object in the next section below for the explanation of virtual splitters.

The Splitters collection has two properties: Count that specifies the number of elements in the collection and Item that returns a member of the collection. See reference for Collection Object in MSDN Library for the description of how to use the properties.

The Controls collection also has one method, IsExist, to determine whether certain index that uniquely identifies a virtual splitter exist in the collection.

 

Splitter Object

A Splitter object in VB Control Manager is a virtual splitter used to represent a real splitter.

VB Control Manager uses PictureBox control as a splitter. Instead of using the PictureBox properties directly, VB Control Manager saves the necessary properties plus several custom properties in virtual splitters. Furthermore, VB Control Manager works with these virtual splitters to manipulate its position, size, etc. and then write it back to the PictureBox controls.

 

The Splitter object has 27 properties as follows:

 

1.   ActiveColor

      Returns/sets the background color used to display the splitter when the user moves it in none live update mode.


2.   BackColor

      Returns/sets the background color used to display the splitter.


3.   Bottom (read-only)

      Returns the distance between the internal bottom edge of the virtual splitter and the top edge of the related Control Manager object.


4.   ClipCursor

      Returns/sets a value that determines whether the mouse pointer is confined to the virtual splitter minimum and maximum x- and y-coordinate when the user moves the splitter.


5.   Enable

      Returns/sets a value that determines whether the splitter is movable.


6.   Height (read-only)

      Returns the height of the virtual splitter.


7.   Id (read-only)

      Returns a value that uniquely identifies the virtual splitter.


8.   IdsCtlBottom (read-only)

      Returns the Id collection of all virtual controls which are effected by the virtual splitter's down-movement.


9.   IdsCtlLeft (read-only)

      Returns the Id collection of all virtual controls which are effected by the virtual splitter's left-movement.


10.  IdsCtlRight (read-only)

      Returns the Id collection of all virtual controls which are effected by the virtual splitter's right-movement.


11.  IdsCtlTop (read-only)

      Returns the Id collection of all virtual controls which are effected by the virtual splitter's up-movement.


12.  IdsSplBottom (read-only)

      Returns the Id collection of all virtual splitters which are effected by the virtual splitter's down-movement.


13.  IdsSplLeft (read-only)

      Returns the Id collection of all virtual splitters which are effected by the virtual splitter's left-movement.


14.  IdsSplRight (read-only)

      Returns the Id collection of all virtual splitters which are effected by the virtual splitter's right-movement.


15.  IdsSplTop (read-only)

      Returns the Id collection of all virtual splitters which are effected by the virtual splitter's up-movement.


16.  Left (read-only)

      Returns the distance between the internal left edge of the virtual splitter and the left edge of the related Control Manager object.


17.  LiveUpdate

      Returns/sets a value that determines whether the controls should be resized as the splitter is moved.


18.  MaxXc (read-only)

      Returns the maximum x-coordinate of the virtual splitter.


19.  MaxYc (read-only)

      Returns the maximum y-coordinate of the virtual splitter.


20.  MinXc (read-only)

      Returns the minimumx-coordinate of the virtual splitter.


21.  MinYc (read-only)

      Returns the minimum y-coordinate of the virtual splitter.


22.  Orientation (read-only)

      Returns the virtual splitter movement direction.


23.  Right (read-only)

      Returns the distance between the internal right edge of the virtual splitter and the left edge of the related Control Manager object.


24.  Top (read-only)

      Returns the distance between the internal top edge of the virtual splitter and the top edge of the related Control Manager object.

 

25.  Width (read-only)

      Returns the width of the virtual splitter.


26.  Xc (read-only)

      Returns the x-coordinate of the virtual splitter center.


27.  Yc (read-only)

      Returns the y-coordinate of the virtual splitter center.
